## 1. 项目概述

### 1.1 项目背景

随着个人开发的小工具数量增加（预期从1个增长到100个），需要一个统一的入口来管理和展示这些工具。同时需要了解各工具的使用情况，以便优化和迭代。

### 1.2 项目目标

1. **统一入口**：提供一个美观、易用的工具导航页面
2. **多端支持**：支持网页端和微信小程序访问
3. **数据洞察**：通过第三方统计服务了解各工具使用情况
4. **低成本运营**：MVP 阶段采用纯静态方案，成本接近零

### 1.3 项目命名

- **项目代号**：SparkInspyer（灵感火花）
- **导航页**：SparkHub

### 1.4 域名规划

| 用途 | 域名格式 | 示例 |
|------|----------|------|
| 导航页 | `主域名` | spark.example.com |
| 各工具 | `工具名.主域名` | tigang.spark.example.com |

## 2. 用户角色

### 2.1 普通用户（访客）

- 浏览工具列表
- 搜索工具
- 查看工具详情和使用说明
- 收藏工具（本地存储）
- 查看最近使用的工具
- 访问并使用各个工具

### 2.2 管理员（开发者本人）

- 编辑 `tools.json` 文件管理工具列表
- 通过百度统计后台查看使用数据
- 部署更新后的导航页

## 3. 功能需求

### 3.1 导航页（SparkHub）

#### 3.1.1 工具列表展示

| 功能点 | 优先级 | 描述 |
|--------|--------|------|
| 工具卡片展示 | P0 | 以卡片形式展示工具，包含名称、图标、简介 |
| 工具详情弹窗 | P0 | 点击卡片展示详细说明、使用指南 |
| 跳转工具 | P0 | 点击"立即使用"跳转到工具页面 |
| 工具搜索 | P0 | 支持按名称、描述搜索工具 |
| 最近使用 | P1 | 展示最近使用的工具（本地存储） |
| 收藏功能 | P1 | 支持收藏工具，收藏列表优先展示 |

#### 3.1.2 工具卡片信息

```
工具卡片
├── 图标（必填）
├── 名称（必填）
├── 简介（必填，一句话描述）
├── 标签（可选，如"新品"、"热门"）
└── 子域名（系统字段，用于跳转）

工具详情
├── 完整描述
├── 使用说明
├── 截图/预览图（可选）
├── 版本信息
└── 更新日期
```

#### 3.1.3 布局与交互

- **响应式布局**：适配手机、平板、桌面端
- **卡片网格**：默认3列（桌面）、2列（平板）、1列（手机）
- **快速访问区**：页面顶部展示"收藏"和"最近使用"
- **搜索框**：顶部固定，支持实时搜索

### 3.2 微信小程序

#### 3.2.1 功能范围

| 功能 | 网页端 | 小程序 | 备注 |
|------|--------|--------|------|
| 工具列表 | ✅ | ✅ | 核心功能 |
| 搜索 | ✅ | ✅ | 核心功能 |
| 收藏 | ✅ | ✅ | 本地存储 |
| 最近使用 | ✅ | ✅ | 本地存储 |
| 工具详情 | ✅ | ✅ | 核心功能 |
| 跳转工具 | 直接跳转 | webview打开 | 小程序需配置业务域名 |

#### 3.2.2 小程序特性

- 首次访问引导用户添加到"我的小程序"
- 支持分享工具到微信好友/朋友圈
- 使用小程序本地缓存存储收藏和最近使用

### 3.3 工具管理（手动）

MVP 阶段通过编辑 `tools.json` 文件管理工具：

#### 3.3.1 管理流程

```
1. 在本地编辑 public/data/tools.json 文件
2. 添加/修改/删除工具信息
3. 提交代码到 GitHub
4. 自动触发 CI/CD 部署
5. 更新后的导航页自动上线
```

#### 3.3.2 工具信息字段

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | string | ✅ | 唯一标识 |
| name | string | ✅ | 工具名称 |
| slug | string | ✅ | 子域名前缀 |
| icon | string | ✅ | 图标（emoji 或图片路径） |
| summary | string | ✅ | 一句话简介 |
| description | string | ❌ | 详细描述（Markdown） |
| guide | string | ❌ | 使用说明（Markdown） |
| screenshots | string[] | ❌ | 截图 URL 列表 |
| version | string | ❌ | 版本号 |
| tags | string[] | ❌ | 标签：new/hot/featured |
| status | string | ✅ | 状态：active/inactive/maintenance |
| sortOrder | number | ✅ | 排序权重 |

### 3.4 数据统计（第三方服务）

使用百度统计作为数据分析平台：

#### 3.4.1 可获取的数据

| 统计指标 | 百度统计功能 | 说明 |
|----------|--------------|------|
| PV/UV | ✅ 流量分析 | 自动采集 |
| 活跃用户 | ✅ 访客分析 | DAU/WAU/MAU |
| 工具点击 | ✅ 事件分析 | 自定义事件上报 |
| 停留时长 | ✅ 页面分析 | 自动采集 |
| 来源渠道 | ✅ 来源分析 | 自动采集 |
| 地域分布 | ✅ 地域分析 | 自动采集 |
| 设备分布 | ✅ 设备分析 | 自动采集 |

#### 3.4.2 自定义事件

在导航页中埋点上报以下事件：

| 事件名 | 触发时机 | 参数 |
|--------|----------|------|
| tool_click | 点击工具卡片 | tool_id |
| tool_enter | 进入工具 | tool_id |
| search | 搜索 | keyword, result_count |
| favorite_add | 添加收藏 | tool_id |
| favorite_remove | 取消收藏 | tool_id |

## 4. 非功能需求

### 4.1 性能要求

| 指标 | 目标 |
|------|------|
| 首屏加载 | < 2秒（CDN 加速） |
| 搜索响应 | < 100ms（本地过滤） |
| 页面大小 | < 500KB（gzip 后） |

### 4.2 兼容性要求

- **浏览器**：Chrome、Safari、Firefox、Edge 最新两个版本
- **移动端**：iOS 12+、Android 8+
- **小程序**：微信基础库 2.20.0+

### 4.3 可扩展性设计

为未来功能预留扩展空间：

| 未来功能 | 预留设计 |
|----------|----------|
| 用户登录 | 数据结构预留 userId 字段 |
| 工具分类 | 数据结构预留 category 字段 |
| 动态管理后台 | 保持数据结构一致，仅需添加 API 层 |
| 自建统计 | 事件埋点格式标准化，便于迁移 |

## 5. 约束与假设

### 5.1 约束条件

1. MVP 阶段为纯静态部署，无后端服务
2. 工具管理通过编辑 JSON 文件完成
3. 统计数据在百度统计后台查看
4. 工具均为 Web 应用，通过子域名访问
5. 小程序内的工具需通过 webview 打开

### 5.2 假设条件

1. 用户已有域名并可配置子域名
2. 已完成 ICP 备案（使用国内 CDN 需要）
3. 已注册百度统计账号
4. 已注册微信小程序账号
