## 1. 项目概述

### 1.1 项目背景

随着个人开发的小工具数量增加（预期从1个增长到100个），需要一个统一的入口来管理和展示这些工具。同时需要了解各工具的使用情况，以便优化和迭代。

### 1.2 项目目标

1. **统一入口**：提供一个美观、易用的工具导航页面
2. **多端支持**：支持网页端和微信小程序访问
3. **数据洞察**：通过管理后台了解各工具使用情况
4. **可扩展性**：为未来的用户体系、分享功能预留扩展空间

### 1.3 项目命名

- **项目代号**：SparkInspyer（灵感火花）
- **导航页**：SparkHub
- **管理后台**：SparkAdmin

### 1.4 域名规划

| 用途 | 域名格式 | 示例 |
|------|----------|------|
| 导航页 | `主域名` | spark.example.com |
| 管理后台 | `admin.主域名` | admin.spark.example.com |
| 各工具 | `工具名.主域名` | tigang.spark.example.com |

## 2. 用户角色

### 2.1 普通用户（访客）

- 浏览工具列表
- 搜索工具
- 查看工具详情和使用说明
- 收藏工具（本地存储）
- 查看最近使用的工具
- 访问并使用各个工具

### 2.2 管理员

- 查看各工具使用数据
- 查看整体数据大盘
- 管理工具信息（增删改）

## 3. 功能需求

### 3.1 导航页（SparkHub）

#### 3.1.1 工具列表展示

| 功能点 | 优先级 | 描述 |
|--------|--------|------|
| 工具卡片展示 | P0 | 以卡片形式展示工具，包含名称、图标、简介 |
| 工具详情弹窗 | P0 | 点击卡片展示详细说明、使用指南 |
| 跳转工具 | P0 | 点击"立即使用"跳转到工具页面 |
| 工具搜索 | P0 | 支持按名称、描述搜索工具 |
| 最近使用 | P1 | 展示最近使用的工具（本地存储） |
| 收藏功能 | P1 | 支持收藏工具，收藏列表优先展示 |

#### 3.1.2 工具卡片信息

```
工具卡片
├── 图标（必填）
├── 名称（必填）
├── 简介（必填，一句话描述）
├── 标签（可选，如"新品"、"热门"）
└── 子域名（系统字段，用于跳转）

工具详情
├── 完整描述
├── 使用说明
├── 截图/预览图（可选）
├── 版本信息
└── 更新日期
```

#### 3.1.3 布局与交互

- **响应式布局**：适配手机、平板、桌面端
- **卡片网格**：默认3列（桌面）、2列（平板）、1列（手机）
- **快速访问区**：页面顶部展示"收藏"和"最近使用"
- **搜索框**：顶部固定，支持实时搜索

### 3.2 微信小程序

#### 3.2.1 功能范围

| 功能 | 网页端 | 小程序 | 备注 |
|------|--------|--------|------|
| 工具列表 | ✅ | ✅ | 核心功能 |
| 搜索 | ✅ | ✅ | 核心功能 |
| 收藏 | ✅ | ✅ | 本地存储 |
| 最近使用 | ✅ | ✅ | 本地存储 |
| 工具详情 | ✅ | ✅ | 核心功能 |
| 跳转工具 | 直接跳转 | webview打开 | 小程序需配置业务域名 |

#### 3.2.2 小程序特性

- 首次访问引导用户添加到"我的小程序"
- 支持分享工具到微信好友/朋友圈
- 使用小程序本地缓存存储收藏和最近使用

### 3.3 管理后台（SparkAdmin）

#### 3.3.1 数据统计

| 统计指标 | 描述 | 优先级 |
|----------|------|--------|
| PV/UV | 页面访问量/独立访客数 | P0 |
| 活跃用户数 | DAU/WAU/MAU | P0 |
| 工具使用频次 | 各工具访问排行 | P0 |
| 使用时长 | 用户在各工具的停留时间 | P1 |
| 访问趋势 | 按日/周/月的访问趋势图 | P0 |
| 来源分析 | 网页端/小程序占比 | P1 |

#### 3.3.2 数据大盘

- **概览卡片**：今日PV、今日UV、活跃工具数、总工具数
- **趋势图表**：访问趋势折线图（支持切换时间范围）
- **工具排行**：按访问量排序的工具列表
- **来源分布**：饼图展示各渠道占比

#### 3.3.3 工具管理

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 工具列表 | 查看所有工具 | P0 |
| 添加工具 | 填写工具信息并发布 | P0 |
| 编辑工具 | 修改工具信息 | P0 |
| 删除工具 | 下线工具 | P1 |
| 排序调整 | 调整工具展示顺序 | P2 |

## 4. 数据埋点

### 4.1 埋点事件

| 事件名 | 触发时机 | 参数 |
|--------|----------|------|
| page_view | 页面加载 | page_type, tool_id |
| tool_click | 点击工具卡片 | tool_id, source |
| tool_enter | 进入工具 | tool_id, from |
| tool_exit | 离开工具 | tool_id, duration |
| search | 搜索 | keyword, result_count |
| favorite_add | 添加收藏 | tool_id |
| favorite_remove | 取消收藏 | tool_id |
| share | 分享 | tool_id, platform |

### 4.2 用户标识

- **当前阶段**：使用设备指纹（deviceId）+ 本地存储（localStorage）
- **未来扩展**：支持微信登录后使用 openId/unionId

## 5. 非功能需求

### 5.1 性能要求

| 指标 | 目标 |
|------|------|
| 首屏加载 | < 2秒 |
| 接口响应 | < 500ms |
| 搜索响应 | < 200ms |

### 5.2 兼容性要求

- **浏览器**：Chrome、Safari、Firefox、Edge 最新两个版本
- **移动端**：iOS 12+、Android 8+
- **小程序**：微信基础库 2.20.0+

### 5.3 可扩展性设计

为未来功能预留扩展空间：

| 未来功能 | 预留设计 |
|----------|----------|
| 用户登录 | 数据库预留 userId 字段 |
| 工具分类 | 数据库预留 category 字段 |
| 使用记录 | 埋点数据支持用户关联 |
| 分享体系 | 分享事件埋点，预留邀请码字段 |

## 6. 约束与假设

### 6.1 约束条件

1. 管理后台仅限内部使用，无需复杂权限系统
2. 初期不做用户登录，所有功能基于设备识别
3. 工具均为 Web 应用，通过子域名访问
4. 小程序内的工具需通过 webview 打开

### 6.2 假设条件

1. 用户已有域名并可配置子域名
2. 各工具已接入统一的埋点 SDK
3. 腾讯云作为部署环境
