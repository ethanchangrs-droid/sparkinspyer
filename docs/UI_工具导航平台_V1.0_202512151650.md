## 1. 设计理念

### 1.1 设计原则

| 原则 | 描述 |
|------|------|
| **简洁优先** | 减少视觉噪音，突出工具本身 |
| **快速触达** | 最多2步到达任意工具 |
| **愉悦体验** | 精致的微交互，提升使用体验 |
| **一致性** | 多端体验保持一致的设计语言 |

### 1.2 设计风格

- **视觉风格**：现代、轻盈、有科技感
- **色彩基调**：深色主题为主，突出工具卡片
- **动效风格**：流畅自然，富有弹性
- **图标风格**：线性图标，圆润边角

### 3.3 页面动效

#### 3.3.1 首屏加载动画

```
序列动画（使用 framer-motion stagger）：

1. Logo 从上方滑入（delay: 0ms）
2. 搜索框从上方滑入（delay: 100ms）
3. Hero 标题淡入 + 从下方滑入（delay: 200ms）
4. 快速访问区块滑入（delay: 300ms）
5. 工具卡片依次淡入（delay: 400ms + i * 50ms）
```

#### 3.3.2 卡片悬浮效果

```typescript
// framer-motion 配置
const cardVariants = {
  initial: { y: 0, boxShadow: 'var(--shadow-card)' },
  hover: {
    y: -4,
    boxShadow: 'var(--shadow-hover)',
    transition: {
      type: 'spring',
      stiffness: 400,
      damping: 25,
    },
  },
  tap: {
    scale: 0.98,
    transition: {
      type: 'spring',
      stiffness: 400,
      damping: 25,
    },
  },
};
```

#### 3.3.3 搜索交互

```
1. 点击搜索框 → 展开搜索面板
2. 输入关键词 → 实时过滤（debounce 200ms）
3. 匹配结果高亮显示
4. 选择结果 → 关闭面板，滚动到卡片或打开详情
```

### 3.4 响应式设计

| 断点 | 宽度 | 列数 | 布局调整 |

| Mobile | < 640px | 1 | 搜索框全宽，卡片堆叠 |
| Tablet | 640-1024px | 2 | 搜索框居中 400px |
| Desktop | > 1024px | 3 | 完整布局 |
| Large | > 1440px | 4 | 更宽的网格 |

## 7. 交互规范

### 7.1 点击反馈

| 元素 | 反馈效果 |
|------|----------|
| 按钮 | 缩放至 0.95 + 颜色加深 |
| 卡片 | 缩放至 0.98 + 阴影增强 |
| 图标按钮 | 缩放至 0.9 + 透明度降低 |
| 链接 | 下划线 + 颜色加深 |

### 7.2 加载状态

| 场景 | 展示方式 |
|------|----------|
| 页面首次加载 | 骨架屏 |
| 数据刷新 | 顶部进度条 |
| 按钮操作 | 按钮内 Spinner |
| 图片加载 | 渐进式淡入 |

### 7.3 空状态

| 场景 | 展示内容 |
|------|----------|
| 无收藏 | 插画 + "还没有收藏的工具" + 引导文案 |
| 无搜索结果 | 插画 + "没有找到相关工具" + 建议 |
| 工具列表为空 | 插画 + "暂无工具" + 添加按钮（管理端） |
